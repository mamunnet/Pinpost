┌───────────────────────────────────────────────────────────────────┐
│  📍 STEP 1: GET FREE DATABASE (5 MIN)                             │
└───────────────────────────────────────────────────────────────────┘

  1. Visit: https://www.mongodb.com/cloud/atlas/register
  2. Sign up with Google (fastest!)
  3. Choose FREE (M0) plan
  4. Click "Create Database Access" → Add user
  5. Autogenerate password → SAVE IT! 📝
  6. Click "Network Access" → Allow from anywhere (0.0.0.0/0)
  7. Click "Database" → Connect → Copy connection string
  8. Replace <password> with your actual password
  9. SAVE THIS STRING! 📝


┌───────────────────────────────────────────────────────────────────┐
│  🖥️  STEP 2: GET HOSTINGER VPS (5 MIN)                            │
└───────────────────────────────────────────────────────────────────┘

  1. Visit: https://www.hostinger.com/vps-hosting
  2. Choose "KVM 2" plan ($5.99/month)
  3. Complete purchase
  4. Go to hPanel → Your VPS
  5. Click "Operating System" tab
  6. Choose "Ubuntu 22.04 with Docker" ⭐
  7. Set root password → SAVE IT! 📝
  8. Click "Change OS" → Wait 10 minutes


┌───────────────────────────────────────────────────────────────────┐
│  🚀 STEP 3: DEPLOY (2 COMMANDS!)                                  │
└───────────────────────────────────────────────────────────────────┘

  In Hostinger Browser Terminal:

  Command 1 (Download):
  cd /root && git clone https://github.com/mamunnet/Pinpost.git && cd Pinpost

  Command 2 (Deploy):
  bash easy-deploy.sh

  Then:
  - Paste your MongoDB connection string
  - Press Enter for auto IP detection
  - Wait 10 minutes
  - DONE! 🎉

┌───────────────────────────────────────────────────────────────────┐
│  📋 DAILY COMMANDS (Copy & Paste in Terminal)                     │
└───────────────────────────────────────────────────────────────────┘

  Go to project:
  cd /root/Pinpost

  View logs:
  docker-compose logs -f

  Check status:
  docker-compose ps

  Restart:
  docker-compose restart

  Stop:
  docker-compose down

  Start:
  docker-compose up -d

  Update:
  git pull && docker-compose up -d --build

  Backup:
  tar -czf backup-$(date +%Y%m%d).tar.gz backend/uploads .env


┌───────────────────────────────────────────────────────────────────┐
│  🆘 QUICK FIXES                                                   │
└───────────────────────────────────────────────────────────────────┘

  Can't access website:
  cd /root/Pinpost && docker-compose ps
  docker-compose up -d

  MongoDB error:
  nano .env  (check MONGO_URL is correct)
  Ctrl+X, Y, Enter to save
  docker-compose restart

  Images won't upload:
  chmod -R 777 backend/uploads
  docker-compose restart

  Containers restarting:
  docker-compose logs backend  (see the error)

  Everything broken:
  cd /root/Pinpost
  docker-compose down
  docker system prune -f
  docker-compose up -d --build
